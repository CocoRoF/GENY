## 결정 / trade-off

- **Class-level `CAPABILITIES` constant** vs `capabilities(input)` method override:
  Class attr 채택. 모든 Geny tool 의 capability 는 input 무관 (concurrency/destructive 가 호출 인자에 따라 안 변함). method override 는 더 풍부하지만 40 tool 에 대해 boilerplate 두 배.
- **Browser tools 전부 `concurrency_safe=False`**:
  단일 Chromium singleton (`_BrowserManager`) 공유 → race risk. screenshot/page_info 는 read_only 지만 같은 페이지 상태에 의존하므로 fan-out 불가.
- **Game tools (`feed`/`play`/`gift`/`talk`) 전부 `concurrency_safe=False`**:
  `MutationBuffer` 가 thread-local 이 아니라 단일 인스턴스. 동시 호출 시 race. talk 만 `idempotent=True` (familiarity bump 두번 = 한번과 동일 narration).
- **`max_result_chars` 명시적 설정**:
  `web_fetch=50_000`, `web_fetch_multiple=100_000`, `web_search=10_000`. browser_screenshot 만 `0` (base64 PNG 는 truncate 하면 깨짐).
